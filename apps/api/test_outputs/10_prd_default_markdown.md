# 默认模板 PRD

## 一、背景与目标

### 执行摘要
本项目旨在为医疗集团构建一套支持多院区部署的全面功能测试平台（默认模板），在严格遵循HIPAA、GDPR、网络安全法及等保三级要求的前提下，实现患者数据的安全互通与临床工作流协同。

### 当前痛点
- 现有系统孤岛导致跨院区患者需重复检查，影响诊疗效率与患者安全。
- 手工权限管理无法满足等保三级对访问控制与最小权限的要求。
- 缺乏全链路审计追踪，合规风险高，难以应对监管审查。
- 多院区数据标准不统一，导致临床决策支持信息缺失或错误。

### 产品愿景
建立安全、可审计、高可用的医疗数据协作中台，以默认模板化方式支撑所有院区快速复制部署，实现“患者不动、数据流动”的安全诊疗模式，确保任何数据访问均可追溯、可审计、可控制。

### 项目范围
**包含范围**:
- 跨院区患者360视图查询与PHI字段级脱敏。
- 基于RBAC的四级粒度访问控制（院区-科室-角色-字段）。
- 全量操作审计日志、数字签名与防篡改校验。
- 等保三级要求的技术对接与管理流程落地。

**不包含范围**:
- AI辅助诊断算法开发。
- 医保结算与财务对账系统。
- 院内IoT硬件设备的直接接入与管理。

## 二、用户故事

### US-001: 主治医师

> 作为跨院区会诊的主治医师，我希望在通过双因素认证后，能够查看授权院区内患者的历史病历与检验结果，以便制定准确的诊疗方案。

**优先级**: P0

**验收标准**:
- 医生登录后通过MFA验证，可在患者360视图中查看已授权院区的患者数据。
- 系统仅显示与医生执业范围相关的PHI字段，非授权字段自动脱敏或隐藏。
- 每次数据查看行为生成包含用户ID、时间戳、IP地址、查看内容摘要的审计日志。
- 会话在15分钟无操作后自动锁定，需重新认证方可继续访问。

### US-002: 系统管理员

> 作为集团IT系统管理员，我希望能按照院区、科室、角色、数据字段四级粒度配置访问权限，并即时生效，以满足最小权限原则。

**优先级**: P0

**验收标准**:
- 管理员可在权限控制台中配置“院区-科室-角色-字段”四级规则。
- 权限变更在60秒内同步至所有院区节点，并触发审计事件记录变更详情。
- 系统支持权限继承检测，当配置冲突时自动告警并阻止生效。
- 管理员的所有配置操作本身也被记录为不可删除的高优先级审计日志。

### US-003: 合规审计员

> 作为合规审计员，我需要能够按多维度筛选并导出不可篡改的审计日志报告，以应对监管机构的HIPAA与等保三级审查。

**优先级**: P1

**验收标准**:
- 支持按时间范围、用户账号、操作类型、数据对象、院区站点五维组合筛选。
- 导出文件（PDF/CSV）附带数字签名与SHA-256哈希值，任何改动均可被校验发现。
- 在线审计日志原始数据保留期不少于6年，且仅只读存储，禁止修改或物理删除。
- 导出审计报告的行为本身会生成新的审计记录，包含下载人、时间与文件指纹。

### US-004: 急诊护士

> 作为急诊护士，在遇到无法立即识别身份的患者时，我希望能通过临时授权码快速解锁急救档案，确保不延误抢救时间。

**优先级**: P1

**验收标准**:
- 急诊场景下，护士可申请一次性临时授权码，解锁无名患者的急救档案视图。
- 临时授权码有效期为30分钟，过期后系统自动回收权限并关闭数据访问。
- 临时访问期间，患者档案界面持续高亮标记“临时授权”状态，并实时通知科室负责人。
- 患者身份确认后，临时访问记录自动迁移至正式患者主索引档案，保持审计连续性。

## 系统建议（AI生成）
1. 各院区现有HIS/EMR系统的数据标准、接口协议（HL7 FHIR/DICOM等）及对接时间表。
2. 集团层面患者主索引(MPI)的匹配规则、冲突解决机制与数据清洗方案。
3. 临床科室对PHI字段的敏感级别分级定义（如公开、内部、机密、高度机密）。
4. 等保三级测评机构的具体技术检测项、渗透测试范围与计划排期。
5. 现有网络拓扑结构、院区间专线带宽、延迟数据及灾备中心部署情况。
6. 法务团队对GDPR跨境传输条款、患者知情同意书模板的最终解释与落地要求。

## PRD标准结构
1. 背景与目标
  - 项目定位与业务价值
  - 多院区协同与患者安全驱动
  - 合规与等保三级要求映射
2. 用户故事
  - 临床医护人员角色画像
  - 系统管理员与合规审计员
  - 跨院区与急诊特殊场景
3. 业务流程
  - 患者数据跨院查询流程
  - RBAC权限申请与审批流
  - 审计日志生成与归档流
4. 功能规格
  - 患者360视图与PHI脱敏
  - 四级粒度RBAC权限引擎
  - 审计追踪中心与不可篡改存储
  - 多院区数据同步与冲突解决
5. 数据需求
  - PHI数据分类与加密策略
  - 患者主索引(MPI)管理
  - 数据留存与销毁策略
6. 合规要求
  - HIPAA/GDPR/网络安全法映射
  - 等保三级技术与管理要求
  - 审计追踪完整性校验
7. 数据埋点
  - 用户行为审计埋点
  - 系统性能与异常监控
  - 合规事件实时告警
8. 里程碑
  - MVP: 单院区核心功能与基础审计
  - M2: 三院区联调与等保测评通过
  - M3: 全面上线与合规验收
