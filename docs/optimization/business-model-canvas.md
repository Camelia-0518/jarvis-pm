## 1. 客户细分 (Customer Segments)

### 1.1 主要细分

| 细分 | 描述 | 规模预估 | 价值 |
|:-----|:-----|:---------|:-----|
| **前端转PM** | 有前端开发背景，正在转型或刚转型产品经理的从业者 | 中国约10万人 | 高（核心用户） |
| **初级PM** | 1-3年经验的产品经理，负责具体功能模块 | 中国约50万人 | 中（扩展用户） |
| **团队Leader** | 产品团队负责人，关注团队效率 | 中国约5万人 | 高（付费能力） |

### 1.2 客户画像

#### 画像A：前端转PM的小明
```yaml
人口统计:
  年龄: 26-30岁
  地点: 二线城市
  收入: 15-25万/年
  教育: 本科，计算机相关专业

行为特征:
  技术栈: Vue/React, JavaScript
  转型阶段: 转型6个月内
  学习习惯: 通过实践学习，喜欢看教程
  社交: 活跃于技术社区（掘金/知乎）

痛点:
  - 写PRD时容易写成技术方案
  - 评审会上被挑战用户价值
  - 不知道产品思维和技术思维的差异
  - 担心转型失败影响职业发展

需求:
  - 有人指导如何写PRD
  - 快速掌握产品思维
  - 获得同行认可
  - 证明自己的能力
```

#### 画像B：初级PM小李
```yaml
人口统计:
```

### 10.2 盈亏平衡分析

```
固定成本: ¥120,000/月
变动成本: ¥20/用户/月
客单价: ¥99/月
付费率: 5%

盈亏平衡点:
设需要X个免费用户
收入 = X × 5% × ¥99 = ¥4.95X
成本 = ¥120,000 + X × ¥20

盈亏平衡时:
¥4.95X = ¥120,000 + ¥20X
X = ¥120,000 / (¥4.95 - ¥20) ≈ 8,000用户

结论：需要约8000个免费用户，400个付费用户达到盈亏平衡
```
