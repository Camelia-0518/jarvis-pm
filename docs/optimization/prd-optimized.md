## 4. 功能详细设计

### 4.1 AI PRD生成器

#### 4.1.1 对话式撰写流程

```
用户输入需求草稿
        ↓
[意图识别] → 判断需求类型(新功能/优化/重构)
        ↓
[信息收集] → AI提问补充缺失信息
        ↓
[结构生成] → 生成PRD大纲
        ↓
[内容填充] → 逐章节AI生成
        ↓
[产品思维检查] → 检测技术语言并提醒
        ↓
[优化打磨] → 用户调整 + AI优化
        ↓
输出: 完整PRD文档
```

#### 4.1.2 PRD标准结构（8章）

| 章节 | 内容 | AI生成方式 | 产品思维检查 |
|:-----|:-----|:-----------|:-------------|
| 1. 背景与目标 | 痛点分析、成功指标 | 基于需求推导 | 检查是否聚焦用户价值 |
| 2. 用户故事 | 5个用户故事+验收标准 | 基于场景生成 | 检查是否以用户为中心 |
| 3. 业务流程 | 主流程+分支+异常 | 基于流程描述生成Mermaid图 | 检查流程完整性 |
| 4. 功能规格 | 前端+后台功能点 | 基于需求拆解 | 检查技术语言 |
| 5. 数据需求 | 实体定义、状态机 | 基于功能推导 | - |
| 6. 合规要求 | 等保/医疗/多院区 | 基于行业模板 | 检查合规完整性 |
| 7. 数据埋点 | 核心指标定义 | 基于目标推导 | - |
| 8. 里程碑 | 时间线规划 | 基于功能估算 | - |

#### 4.1.3 产品思维引导设计

**技术语言检测规则**：

```yaml
detect_patterns:
  - pattern: "Vue|React|Angular"
    severity: warning
    message: "检测到前端框架名称，建议描述用户价值而非技术实现"
    suggestion: "将'使用Vue实现'改为'用户可以通过界面快速完成操作'"
    
  - pattern: "API接口|数据库表|字段"
    severity: warning
    message: "检测到技术实现细节，建议聚焦用户需求"
    suggestion: "将'需要设计用户表'改为'系统需要记录用户基本信息'"
    
  - pattern: "并发|缓存|队列"
    severity: info
    message: "技术优化建议可放在非功能需求中"
    suggestion: "将性能要求放在'非功能需求'章节"
```

**产品思维引导示例**：

```
用户输入：
"这个功能需要用Vue实现一个表单，包含姓名、手机号、地址字段，
数据存储到MySQL的用户表"

AI检测并提醒：
⚠️ 检测到技术实现语言

建议修改为：
"作为患者，我希望在借阅申请页面填写个人信息，
以便医院核实我的身份并寄送切片"

验收标准：
- 可输入姓名、手机号、邮寄地址
- 系统自动验证手机号格式
- 地址支持省市区选择"
```
