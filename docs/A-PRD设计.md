## 1. 产品概述

### 1.1 一句话描述

**"一个AI原生的产品管理协作平台，帮助产品经理用对话方式10分钟完成原本需要2天的PRD撰写和评审准备"**

### 1.2 产品定位

| 维度 | 定位 |
|:-----|:-----|
| 目标用户 | 产品经理 (前端转PM/初级PM/中级PM) |
| 核心场景 | PRD撰写、需求评审准备、站会报告生成 |
| 差异化 | AI-First设计，全流程AI辅助 |
| 竞品回避 | 不做通用项目管理，专注PM专属场景 |

### 1.3 核心价值主张

```
传统方式                          Jarvis PM
─────────                        ─────────
PRD撰写: 2-3天        →          10分钟
评审准备: 1天         →          5分钟
站会报告: 15分钟      →          1分钟
合规检查: 人工核对    →          自动扫描
```

## 2. 用户画像

### 2.1 主要用户: 前端转产品经理

| 维度 | 描述 |
|:-----|:-----|
| **用户是谁** | 有前端开发背景，刚转型或正在转型产品经理的从业者 |
| **痛点是什么** | 1. 缺乏PRD写作经验，不知道标准结构<br>2. 容易陷入技术实现细节，忽略用户需求<br>3. 评审准备耗时，担心遗漏风险点<br>4. 多院区/合规场景经验不足 |
| **现有解决方案** | 用Word/飞书文档写PRD，手动整理评审材料 |
| **为什么需要Jarvis PM** | AI引导式PRD撰写，自动提醒产品思维，一键生成评审材料 |

### 2.2 次要用户: 初级产品经理

| 维度 | 描述 |
|:-----|:-----|
| **用户是谁** | 1-3年经验的产品经理，负责具体功能模块 |
| **痛点是什么** | 重复性工作多，PRD模板化严重，创新时间少 |
| **现有解决方案** | 复制粘贴历史PRD，手动修改 |
| **为什么需要Jarvis PM** | 智能复用历史经验，自动化重复工作，聚焦创新 |

### 2.3 用户故事地图

```
作为产品经理，我希望...                                以便...
────────────────────────────────────────────────────────────────────────
[发现]

作为PM，我希望AI能帮我优化口语化需求              转化为标准PRD

[定义]
作为PM，我希望能用对话方式补充需求细节             不用写长文档
作为PM，我希望系统自动检查合规要求                避免遗漏医疗等规范

[评审]

作为PM，我能获得各相关方的预设Q&A                提前准备答复

[跟进]
```

## 4. 功能详细设计

### 4.1 AI PRD生成器

#### 4.1.1 对话式撰写流程

```
用户输入需求草稿
        ↓
[意图识别] → 判断需求类型(新功能/优化/重构)
        ↓
[信息收集] → AI提问补充缺失信息
        ↓
[结构生成] → 生成PRD大纲
        ↓
[内容填充] → 逐章节AI生成
        ↓
[优化打磨] → 用户调整 + AI优化
        ↓
输出: 完整PRD文档
```

#### 4.1.2 PRD标准结构 (8章)

| 章节 | 内容 | AI生成方式 |
|:-----|:-----|:-----------|
| 1. 背景与目标 | 痛点分析、成功指标 | 基于需求推导 |
| 2. 用户故事 | 5个用户故事+验收标准 | 基于场景生成 |
| 3. 业务流程 | 主流程+分支+异常 | 基于流程描述生成Mermaid图 |
| 4. 功能规格 | 前端+后台功能点 | 基于需求拆解 |
| 5. 数据需求 | 实体定义、状态机 | 基于功能推导 |
| 6. 合规要求 | 等保/医疗/多院区 | 基于行业模板 |
| 7. 数据埋点 | 核心指标定义 | 基于目标推导 |
| 8. 里程碑 | 时间线规划 | 基于功能估算 |

## 6. 数据模型

### 6.1 核心实体

```typescript
// 用户
interface User {
  id: string;
  email: string;
  name: string;
  avatar?: string;
  role: 'admin' | 'member' | 'guest';
  preferences: UserPreferences;
  createdAt: Date;
}

// 工作空间
interface Workspace {
  id: string;
  name: string;
  slug: string;
  ownerId: string;
  members: WorkspaceMember[];
  plan: 'free' | 'pro' | 'enterprise';
  createdAt: Date;
}

// 项目
interface Project {
  id: string;
  workspaceId: string;
  name: string;
  description?: string;
  industry: 'medical' | 'ecommerce' | 'saas' | 'other';
  status: 'active' | 'archived' | 'deleted';
  settings: ProjectSettings;
  createdAt: Date;
  updatedAt: Date;
}

// PRD文档
interface PRD {
  id: string;
  projectId: string;
  title: string;
  version: string;
  status: 'draft' | 'review' | 'approved' | 'implemented';
  content: PRDContent;  // 结构化内容
  markdown: string;     // 原始Markdown
  aiGenerated: boolean;
  createdBy: string;
  createdAt: Date;
  updatedAt: Date;
}

// PRD内容结构
interface PRDContent {
  background: Section;
  userStories: UserStory[];
  businessFlow: Section;
  functionalSpecs: Section;
  dataRequirements: Section;
  compliance: Section;
  analytics: Section;
  milestones: Section;
}

// 用户故事
interface UserStory {
  id: string;
  role: string;
  action: string;
  benefit: string;
  acceptanceCriteria: string[];
  priority: 'P0' | 'P1' | 'P2';
}

// 版本历史
interface Version {
  id: string;
  documentId: string;
  version: number;
  content: string;
  changes: ChangeSummary;
  createdBy: string;
  createdAt: Date;
}

// 评审材料
interface ReviewMaterial {
  id: string;
  prdId: string;
  type: 'agenda' | 'decisions' | 'risks' | 'qa' | 'standup';
  content: object;
  generatedBy: 'ai' | 'manual';
  createdAt: Date;
}

// 模板
interface Template {
  id: string;
  name: string;
  category: 'medical' | 'ecommerce' | 'saas' | 'general';
  structure: PRDContent;
  isBuiltin: boolean;
  createdBy?: string;
}
```

## 8. AI功能设计

### 8.1 Prompt优化层

```typescript
// 自动优化用户输入
function optimizePrompt(input: string, context: Context): StructuredPrompt {
  // 1. 意图识别
  const intent = classifyIntent(input);
  
  // 2. 模板匹配
  const template = matchTemplate(intent, context.industry);
  
  // 3. 上下文加载
  const enrichedContext = loadContext(context);
  
  // 4. 生成结构化prompt
  return {
    role: 'user',
    content: buildPrompt(input, template, enrichedContext)
  };
}

// 优化示例
输入: "帮我看看这个函数为啥跑得慢"
  ↓
输出: {
  task: "代码性能分析",
  target: "函数",
  goal: "找出性能瓶颈",
  dimensions: ["时间复杂度", "空间复杂度", "算法效率"],
  nextStep: "请提供代码或文件路径"
}
```

### 8.2 AI Agent设计

| Agent | 职责 | 触发条件 |
|:-----|:-----|:---------|
| **PRD Agent** | 生成PRD大纲、章节内容、质量检查 | /start prd |
| **Review Agent** | 生成评审材料、识别风险、准备Q&A | /prep review |
| **Standup Agent** | 汇总进度、识别阻塞、生成报告 | /standup |
| **Compliance Agent** | 检查医疗/等保/多院区合规 | /check compliance |

### 8.3 质量检查规则

```yaml
checks:
  - name: "用户故事完整性"
    rule: "PRD必须包含至少3个用户故事，每个有验收标准"
    severity: error
    
  - name: "成功指标定义"
    rule: "必须定义可量化的成功指标"
    severity: error
    
  - name: "医疗合规检查"
    rule: "涉及患者数据必须包含隐私保护说明"
    severity: error
    when: "industry == 'medical'"
    
  - name: "技术语言检测"
    rule: "PRD中不应出现具体技术实现细节"
    severity: warning
    pattern: ["Vue", "React", "API接口", "数据库表"]
    
  - name: "多院区适配"
    rule: "多院区项目需标注各地特殊要求"
    severity: warning
    when: "multiSite == true"
```
