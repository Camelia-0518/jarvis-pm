# 切片借阅平台 PRD 战役计划

> 目标：用5天时间完成高质量PRD，通过评审，建立PM专业形象

---

## 战役总览

```
Day 1-2: 深度调研（建立业务理解）
```

## Day 1 - 实地调研

### 上午：病理科现场观察（2小时）

**目标**：了解真实业务流程，发现痛点

**任务清单**：
- [ ] 观察患者现场申请流程（记录每个步骤耗时）
- [ ] 观察病理科审核流程
- [ ] 拍照记录（申请表单、审核界面、存放区域）
- [ ] 记录各环节等待时间

**观察要点**：
```
患者动线：
到达医院 → 找到病理科 → 取号/排队 → 提交申请 → 等待审核 → 缴费 → 领取切片

记录每个环节的：
- 平均等待时间
- 是否需要往返
- 患者情绪状态
- 工作人员操作复杂度
```

**输出物**：
- 现场照片（5-10张）
- 流程观察记录表
- 初步痛点清单

---

### 下午：患者访谈（2-3人，每人30分钟）

**访谈对象选择**：
- 1位年轻患者（会用手机）
- 1位中老年患者（可能不熟悉线上操作）
- 1位代办人（帮家人办理）

**使用工具**：UserResearchToolkit 患者访谈提纲

**核心问题**：
1. "您是怎么知道需要借阅切片的？"
2. "来之前您知道需要带什么材料吗？"
3. "整个流程中，最让您困扰的环节是什么？"
4. "如果可以在手机上申请，您愿意尝试吗？担心什么？"
5. "您希望多久能拿到切片？"

**输出物**：
- 访谈录音/笔记
- 患者痛点汇总
- 用户需求优先级

## 评审现场策略

### 你的角色定位
你不是"汇报者"，你是**"会议主持人"**。

### 时间控制
- 提前5分钟到场，测试投影
- 严格控制每个章节时间
- 有人跑题时温和拉回："这个问题很好，我们记下来会后再详细讨论，先继续看下一个环节"

### 应对质疑
**被挑战时**：
- 不要 defensive
- 先认可对方观点："您说得对，这个确实需要考虑"
- 提供数据或方案："我调研发现..."或"我建议这样处理..."

**不会回答时**：
- 诚实承认："这个问题我需要再调研一下"
- 给出时间："明天下班前给您答复"
- 记录下来，不要硬编

### 推动决策
- 每个决策点主动询问："关于押金金额，各位的意见是？"
- 有人犹豫时："建议我们先按XX方案试行，上线后根据数据调整"
- 当场确认待办："那今天的结论是XX，待办是YY，负责人ZZ"
